Chapter 2

DSPy Trials Against the Trimodal Reward

This chapter is a logbook entry rather than a polished narrative. The objective was to test whether the new trimodal embedding geometry could act as a reward signal for prompt optimization, using DSPy and Gemini Embedding 2 Preview instead of the earlier loose-fit text heuristics.

The result was mixed in the most useful possible way: one tiny split improved, a harder split did not. That makes this chapter worth publishing in full detail, because it shows the signal is real enough to move while still too fragile to trust blindly.

Experiment Goal

Can DSPy use our new embedding geometry to improve ASCII-art prompting?

Earlier DSPy work in this project optimized prompts using loose textual similarity to dataset examples. This experiment swapped that reward out for a multimodal one. For every generated candidate, we embedded the ASCII as text, rasterized it and embedded it as image, and embedded the interleaved combination as image+text.

The scoring function then asked two things:

  • How well does the generated trimodal triangle align with the target example's trimodal triangle?
  • How self-consistent is the generated triangle internally?
reward = 0.7 * target_trimodal_alignment + 0.3 * generated_self_coherence

In practice:

  • target_trimodal_alignment = mean cosine similarity between generated and target vectors for text, image, and fused
  • generated_self_coherence = mean cosine similarity across the generated text↔image, text↔image+text, and image↔image+text pairs

The best instruction was notably more specific than the default. It emphasized multi-line structure, standard keyboard characters, no surrounding commentary, and readable shape. 

Create a recognizable multi-line ASCII drawing of the requested subject using only standard keyboard characters. The output should be only the ASCII art itself, with no title, labels, explanation, markdown, or surrounding commentary. Make the depiction visually clear and stylistically coherent, choosing a simple but expressive composition that matches the subject category. Use spacing and line breaks carefully to preserve alignment, symmetry, and overall shape. If the subject is plural or category-like, it is acceptable to depict multiple related objects in one scene. Favor clean, readable ASCII art over overly complex detail.

On this split, the optimizer returned the default instruction as the best program: 

Generate ASCII art of the given subject.

That matters. It suggests the reward is not so loose that DSPy can trivially improve it with extra prompt ceremony. The harder split appears to demand either a cleaner benchmark, a stronger metric, or both.

What Worked

Reasons for optimism

  • The reward worked inside DSPy without any LLM judge.
  • The Gemini embedding cache made repeated evaluation practical.
  • Trial A produced a measurable improvement rather than noise alone.
  • The winning prompt on Trial A qualitatively matched the manifold intuition: clarity, shape, spacing, and readable structure.

What Failed

Reasons not to overclaim

  • The more varied Trial B produced no gain at all.
  • The benchmark splits are still tiny.
  • The buildings corpus still contains noisy subjects and generic captions.
  • The reward does not yet normalize for difficulty across categories like maps, monuments, and industrial scenes.

Interpretation

The trimodal reward is promising, but not yet robust.

If both trials had failed, the reward would look too weak to be useful. If both had improved cleanly, the reward would already look ready for heavier optimization. Instead we got the more interesting middle case.

The small 2/2 split shows that trimodal geometry can provide a meaningful direction of improvement. The 4/3 split shows that the same direction does not yet hold up under even modest variation. That makes the right next move fairly clear: improve the benchmark and reward quality before escalating to anything larger than DSPy prompt search.

Next Steps

What this experiment unlocked

  • clean the buildings subjects before more DSPy trials
  • construct train/dev splits that deliberately mix low-area and high-area cases
  • add a small structural guardrail so semantically lucky but visually poor outputs do not score too highly
  • compare plain DSPy prompting against retrieval-augmented prompting from the manifold itself
